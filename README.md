# Simulated Public Revenue Leakage Detection System

## Project Overview
This project simulates a government grade data system designed to detect tax leakage, revenue fraud patterns, under reporting and inconsistencies in public revenue collections using structured SQL analytics.

It models how an internal revenue agency would audit transactions across multiple tax sources, reconcile payments and detect anomalies in compliance behavior.

This is the type of system used in tax authorities, financial crime units, and public accountability dashboards.

## Project Objective
This project simulates a government level tax intelligence system designed to detect:
- Revenue leakage
- Tax underpayment
- Compliance failure
- Duplicate taxpayer identities

It demonstrates advanced SQL skills used in financial auditing and public sector analytics

## Problem Statement
Tax revenue systems often suffer from:
- Fragmented records
- Underpayment of taxes
- Weak compliance tracking
- Duplicate taxpayer identities

This system builds a structured SQL framework to detect and analyze these issues

## Tools Used
- SQL (MySQL Workbench)
- Relational database design

## Key Insights Generated
- Total revenue leakage per tax category
- Compliance rates per taxpayer
- High-risk underpayment detection
- Duplicate taxpayer identification
 -Enforcement prioritization list
